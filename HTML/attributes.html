HTML Attributes
  • 모든 HTML 요소들은 속성을 가질 수 있다.
  • 속성은 요소에 대해서 추가적인 정보를 제공한다.
  • 속성들은 항상 시작 태그 안에서 정의된다.
  • 속성들은 보통 이름/값 쌍의 순서로 작성된다.: 속성명="속성값"
The href Attribute

HTML 링크들은 <a>태그로 정의된다. 링크 주소는 href 속성에서 정의된다.

<a href="https://www.w3schools.com">This is a link</a>
The src Attribute

HTML 이미지들은 <img>태그로 정의된다.
소스 이미지의 파일이름은 src 속성 안에서 정의된다.

The width and height Attributes

HTMl 이미지들은 또한 width와 height 속성들을 가지고 있는 데, 이는 미지의 너비와 높이를 정의한다.
너비와 높이는 기본적으로 픽셀 단위로 나타난다.

The alt Attribute

alt 요소는 이미지를 나타낼 수 없을 때, 사용되는 대체 텍스트이다. alt 속성의 값은 스크린 리더(screen reader)에 의해서 읽어질 수 있다. 이러한 방식으로 누군가 웹페이지를 '읽는다면', 예를 들어서 시각 장애가 있는 사람들은 요소를 '들을 수' 있다.

The style Attribute

style 요소는 색상, 폰트, 크기 등과 같은 요소의 스타일을 정의할 때 사용된다.

The lang Attribute

문서의 사용된 언어는 <html> 태그 안에서 선언할 수 있다.
언어는 lang 속성과 함께 선언된다.
언어를 선언하는 것은 접근성 도구(스크린 리더)와 검색 엔진에 중요하다.

The title Attribute

<p> 요소 안에 title 속성을 추가하면 타이틀 속성의 값이 마우스가 문서 위에 있을 때 툴팁으로 나타난다.

We Suggest: Use Lowercase Attributes

HTML5 표준은 소문자 속성명을 필요로 하지는 않는다.
타이틀 속성은 title 혹은 TITLE과 같이 대소문자로 씌일 수 있다.
W3C는 HTMl에서는 소문자를 쓸 것을 권장하고 있고, XHTML과 같은 엄격한 문서 형식들은 소문자를 사용할 것을 요구한다.

We Suggest: Quote Attribute Values

HTMl5 표준은 속성값 주변에 쌍따옴표를 필요로 하지 않는다.
href 속성은 쌍따옴표 없이 쓸 수 있다.
W3C는 HTML에서 쌍따옴표를 쓸 것을 권고하고 있고, XHTML과 같은 엄격한 문서 형식들은 쌍따옴푤르 사용할 것을 요구한다.
때로는 쌍따옴표를 써야할 필요가 있다. 타이틀 속성의 값에 공백이 있을 경우 속성이 적용되지 않는다.

Single or Double Quotes?
속성값 주변에 쌍따옴표를 쓰는 것은 HTML에서 가장 흔한 경우이나, 따옴표 역시도 사용될 수 있다.
몇 몇 상황에서 속성값 자체가 쌍따옴표를 포함하고 있을 경우, 따옴표를 써야한다.
Chapter Summary
  • 모든 HTMl 요소들은 속성을 가질 수 있다.
  • title 속성은 추가적인 '툴팁' 정보를 제공한다.
  • href 속성은 링크에 대한 주소 저보를 제공한다.
  • width와 height 속성들은 이미지의 크기 정보를 제공한다.
  • alt 속성은 스크린 리더에게 텍스트를 제공한다.
  • 보통 속성명은 소문자를 사용한다.
  • 속성값을 쓸 때는 쌍따옴표를 사용한다.
